# Enterprise Agents

## Идея

`main` работает как директор. Пользователь дает задачу директору. Директор
определяет ответственный отдел и делегирует задачу субагенту. Если субагенту
нужна экспертиза другого отдела, он спрашивает его напрямую через inter-agent
tools и возвращает директору уже собранный ответ.

## Поток работы

```text
Пользователь -> Директор -> Ответственный отдел
                         -> Другой отдел при необходимости
                         <- Сводный ответ ответственного отдела
Пользователь <- Директор
```

## Роли

| Agent | Отдел | Главная зона |
| --- | --- | --- |
| `main` | Директор | постановка задач, маршрутизация, финальный ответ |
| `marketing` | Маркетинг | стиль, посты, публикации, расписание |
| `legal` | Юрист | договоры, ГК РФ, риски организации |
| `technical-director` | Технический директор | нормативка, котлы, комплектации, сдача объекта |
| `production` | Производство | производственные этапы, склад, узкие места |
| `sales-lead` | Продажи | звонки, транскрипты, скрипты, рекомендации |

## Inter-Agent Tools

Субагенты общаются через:

```bash
python3 workspace/tools/agent_tools/ask_agent.py TARGET_AGENT "message"
python3 workspace/tools/agent_tools/ask_agent_async.py TARGET_AGENT "message"
```

Синхронный вариант нужен для коротких консультаций. Асинхронный вариант нужен
для длинной работы, например проверки договора или анализа звонков.

## Источники

Заполните `client-files/data-sources.example.json` и перенесите реальные
значения в локальную конфигурацию runtime. Минимальные source-зоны:

- `client-files/marketing/` - экспорт Telegram/MAX, YouTube, сайт
- `client-files/legal/` - договоры и политика согласования
- `client-files/technical/` - NotebookLM/RAG exports, нормативка, комплектации
- `client-files/production/` - производственные CSV/XLSX
- `client-files/sales/` - звонки, транскрипты, скрипты

## Recurring Jobs

Периодические задачи заводятся через cron tools:

```bash
python3 workspace/tools/cron_tools/cron_add.py
```

Типовые задачи:

- маркетинг: подготовить посты на неделю и черновики для Telegram/MAX
- продажи: ежедневно выгрузить звонки, транскрибировать и оценить по скриптам
- производство: ежедневно проверить узкие места по свежему файлу статуса

## Safety

- Не публиковать сообщения без утверждения.
- Не отправлять договорные правки контрагенту без утверждения.
- Не выгружать записи звонков во внешние сервисы без разрешения.
- Не придумывать факты из неподключенных источников.
